How to Contribute

A few guidelines to keep in mind:
  • Please file issues on GitHub. Make sure your bug reports contain enough details so they can be easily understood by others and quickly fixed.
  • The best way to contribute code is to send a pull request on GitHub.
  • Alternatively, you can send a plain-text patch to the mailing list.
  • Before your code changes can be included, please file the Contribution License Agreement below.
  • Unlike, say, the Apache Software Foundation, we do not require every single code change to be attached to an issue. Feel free to send as many small fixes as you want.
  • Please break down your changes into as many small commits as possible.
  • Please respect the coding style of the code you're changing.

Contribution License Agreement

All contributors are required to file the User Contribution License Agreement below. This agreement is pretty standard and is used by many open-source projects and organizations such as Netty or the Apache Software Foundation, just to cite a couple.

It only takes a couple minutes to file this form electronically.
