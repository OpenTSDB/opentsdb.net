OpenTSDB 2.0 Roadmap

The OpenTSDB community has been growing and many developers have contributed patches, new features and ideas to the project. We are proud to start development on version 2.0 that will integrate many of these features in the official repository. 2.0 will maintain backwards compatiblity with the previous relesae though there maybe some slight changes in the HTTP and JAVA APIs. Below is a roadmap for the next release:

Timeline

  • July 2013 - Initial Release Candidate
  • August 2013 - Release Candidate 2
  • October 2013 - Official Release

We'll do our best to hit these dates and can certainly use all of the help we can get.

Improvements

These are just some of the improvements on the planning board:

  • Improved UID assignment
  • Faster query speeds for complex or aggregated queries
  • More aggregators
  • Graphing options
  • New website and documentation including a user guide

New Features

Some of the new features we're planning to integrate include:
  • A configuration file for storing properties
  • Full RESTful HTTP API that provides access to all of OpenTSDBs features including storing data points via HTTP
  • Plugable data formatters that can accept or emit data in various formats. The default will be JSON though implementations could be created for XML, ProtoBuf or others
  • Annotations that allow tracking of events associated with a particular timeseries or global notes tied to a timestamp
  • Metadata for timeseries, metrics, tag names and tag values so you can find out what is stored in OpenTSDB and add your own notes
  • Metadata trees that organize timeseries into a directory like structure
  • Plugable search abstraction layer to publish metadata and annotations to your search engine of choice, such as Elastic Search
  • Plugable data abstraction layer that will allow the use of data stores other than HBase. Initially we'll release a Cassandra plugin
  • Millsecond timestamp support, enabled as an option for users who need greater resolution
  • Plugable real-time publisher sink to push incoming data out to subscribes such as RabbitMQ or Memcache
  • Debian and FreeBSD packages with init scripts

Requests

Have a feature you would like to see in 2.0? Check the Issues list to see if someone is working on it or already posted a request. Also post a notice to the Users Group

Contributions

We need all of the help we can get so post your thoughts to the users group, then read the Development guid and Contributing page. Create a fork in Github and publish your changes to the "next" branch. Issue a pull request and we'll try and fit it in.
